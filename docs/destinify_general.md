# Destinify: Plataforma de turismo inteligente personalizado

## 1. Descripción

Plataforma de turismo con IA para la gestión personalizada de turismo del Ecuador.

---

## 2. Presentación, necesidades, problemas, público y enfoque

### 📌 Presentación

Destinify es una plataforma de turismo inteligente personalizado que gestiona de forma integral viajes, hoteles, gastronomía, tours, tiempo, preferencias y experiencias en tiempo real mediante inteligencia artificial.

Ayuda al viajero a planificar y gestionar sus viajes de manera eficiente. El sistema analiza preferencias como intereses, presupuesto, idioma y estilo de viaje para recomendar destinos, actividades, hospedaje y experiencias adaptadas a cada perfil.

Además:

- Automatiza la creación de itinerarios
- Organiza reservas
- Centraliza la información del viaje

Durante el viaje:

- Ofrece alertas y recordatorios
- Brinda recomendaciones contextuales
- Ajusta planes según clima, ubicación o disponibilidad

La plataforma aprende del comportamiento del usuario para mejorar continuamente la experiencia.

---

### 📌 Necesidades detectadas

- Falta de automatización en la planificación de viajes
- Dificultad para agendar actividades, hospedaje y gastronomía
- Falta de control del itinerario
- Ausencia de un guía o asistente turístico 24/7

---

### 📌 Problemas que resuelve

#### 1. Falta de personalización en la planificación de viajes

Muchos viajeros reciben recomendaciones genéricas que no se adaptan a sus intereses.

**Solución:**
Uso de IA para generar recomendaciones personalizadas.

#### 2. Dificultad para gestionar todo el viaje en un solo lugar

Uso de múltiples plataformas genera desorden y confusión.

**Solución:**
Centralización de itinerarios, reservas y actividades en una sola plataforma con asistencia en tiempo real.

## 4.	Requisitos funcionales por rol de usuario
### 1.	Rol:  Viajeros independientes

Los viajeros independientes buscan planificar y gestionar sus viajes de forma autónoma, con flexibilidad y personalización.

-Rf1: El sistema debe permitir al usuario registrarse e iniciar sesión.
-Rf2: El sistema debe permitir crear y editar su perfil (intereses, presupuesto, idioma, estilo de viaje).
-Rf3: El sistema debe permitir buscar destinos mediante filtros (ubicación, precio, tipo de experiencia).
-Rf4: El sistema debe generar recomendaciones personalizadas basadas en preferencias.
-Rf5: El sistema debe permitir crear itinerarios manuales Y automáticos con IA.
-Rf6: El sistema debe permitir editar el itinerario en cualquier momento.
-Rf7: El sistema debe mostrar información detallada de destinos (precios, horarios, ubicación, reseñas).
-Rf8: El sistema debe permitir realizar reservas (hoteles, tours, transporte).
-Rf9: El sistema debe centralizar reservas en un panel único.
-Rf10: El sistema debe enviar alertas y recordatorios.
-Rf11: El sistema debe ofrecer recomendaciones en tiempo real durante el viaje.
-Rf12: El sistema debe permitir calificar y comentar servicios.
-Rf13: El sistema debe permitir usar un asistente virtual.
-Rf14: El sistema debe permitir guardar favoritos.
-Rf15: El sistema debe ser accesible desde dispositivos móviles y web.
### 2.	Rol: Turistas organizados

Usuarios que prefieren paquetes completos y planificación automatizada.

-Rf1: El sistema debe permitir registrarse e iniciar sesión.
-Rf2: El sistema debe ofrecer paquetes turísticos completos (viaje, hospedaje, actividades).
-Rf3: El sistema debe permitir seleccionar paquetes según presupuesto y preferencias.
-Rf4: El sistema debe generar itinerarios completos automáticamente.
-Rf5: El sistema debe permitir visualizar el itinerario detallado día por día.
-Rf6: El sistema debe permitir reservar paquetes completos en una sola acción.
-Rf7: El sistema debe permitir pagos integrados.
-Rf8: El sistema debe enviar confirmaciones de reservas.
-Rf9: El sistema debe permitir modificaciones limitadas del itinerario.
-Rf10: El sistema debe ofrecer soporte mediante asistente virtual.
-Rf11: El sistema debe enviar recordatorios automáticos.
-Rf12: El sistema debe permitir calificar la experiencia completa.
### 3.	Rol: Viajeros frecuentes

Usuarios con experiencia que buscan rapidez y eficiencia.

-Rf1: El sistema debe permitir acceso rápido mediante autenticación.
-Rf2: El sistema debe recordar preferencias y viajes anteriores.
-Rf3: El sistema debe permitir duplicar itinerarios anteriores.
-Rf4: El sistema debe generar recomendaciones rápidas basadas en historial.
-Rf5: El sistema debe permitir reservas rápidas (1 clic o similares).
-Rf6: El sistema debe mostrar historial de viajes.
-Rf7: El sistema debe permitir gestionar múltiples viajes simultáneamente.
-Rf8: El sistema debe enviar notificaciones en tiempo real.
-Rf9: El sistema debe permitir acceso a comprobantes y facturas.
-Rf10: El sistema debe permitir evaluar servicios utilizados.
### 4.	Rol: Turistas digitales

Usuarios abiertos al uso de tecnología e inteligencia artificial.

-Rf1: El sistema debe permitir interacción con un asistente conversacional (chat IA).
-Rf2: El sistema debe generar recomendaciones dinámicas en tiempo real.
-Rf3: El sistema debe adaptarse al contexto (ubicación, clima, hora).
-Rf4: El sistema debe permitir comandos por voz o texto.
-Rf5: El sistema debe ofrecer traducción básica integrada.
-Rf6: El sistema debe mostrar mapas interactivos.
-Rf7: El sistema debe permitir integración con otros servicios digitales.
-Rf8: El sistema debe enviar sugerencias inteligentes durante el viaje.
-Rf9: El sistema debe aprender del comportamiento del usuario.
-Rf10: El sistema debe ofrecer experiencia tipo app (PWA o similar).
### 5.	Rol: Administrador de plataforma

Encargado de la gestión global del sistema.

-Rf1: El sistema debe permitir al administrador iniciar sesión con privilegios.
-Rf2: El sistema debe permitir gestionar usuarios (crear, editar, eliminar).
-Rf3: El sistema debe permitir gestionar contenido turístico (destinos, actividades, servicios).
-Rf4: El sistema debe validar y aprobar contenido de proveedores.
-Rf5: El sistema debe gestionar reportes y métricas del sistema.
-Rf6: El sistema debe monitorear el funcionamiento de la plataforma.
-Rf7: El sistema debe gestionar incidencias y soporte.
-Rf8: El sistema debe configurar parámetros del sistema.
-Rf9: El sistema debe gestionar seguridad y permisos.
-Rf10: El sistema debe visualizar estadísticas de uso.
### 6. Rol: Proveedor de servicios turísticos

Incluye hoteles, restaurantes, agencias, etc.

-Rf1: El sistema debe permitir registrarse como proveedor.
-Rf2: El sistema debe permitir crear y gestionar servicios (hoteles, tours, restaurantes).
-Rf3: El sistema debe permitir editar información (precios, horarios, disponibilidad).
-Rf4: El sistema debe permitir subir imágenes y descripciones.
-Rf5: El sistema debe gestionar reservas recibidas.
-Rf6: El sistema debe confirmar o rechazar solicitudes de reserva.
-Rf7: El sistema debe visualizar historial de reservas.
-Rf8: El sistema debe responder a reseñas de usuarios.
-Rf9: El sistema debe ver métricas de rendimiento.
-Rf10: El sistema debe recibir notificaciones de nuevas reservas.

### 7. Rol: Guía turístico

Encargado de ofrecer experiencias y tours personalizados.

-Rf1: El sistema debe permitir registrarse como guía turístico.
-Rf2: El sistema debe permitir publicar tours y actividades.
-Rf3: El sistema debe definir disponibilidad y horarios.
-Rf4: El sistema debe gestionar reservas de sus tours.
-Rf5: El sistema debe comunicarse con los viajeros.
-Rf6: El sistema debe mostrar perfil del guía con calificaciones.
-Rf7: El sistema debe permitir recibir evaluaciones de usuarios.
-Rf8: El sistema debe enviar notificaciones de reservas.
-Rf9: El sistema debe permitir cancelar o reprogramar actividades.
-Rf10: El sistema debe visualizar historial de servicios prestados.

### 8. Restricciones por rol de usuario

## 1. Rol: Viajero independiente

Restricciones:
El usuario debe registrarse para acceder a funciones avanzadas.
El usuario no puede modificar información de otros usuarios.
El usuario solo puede gestionar sus propias reservas e itinerarios.
El usuario depende de conexión a internet para funciones en tiempo real.
Algunas reservas pueden depender de disponibilidad de terceros.
El usuario no puede publicar servicios turísticos en la plataforma.
El usuario no tiene permisos administrativos.
El usuario no puede alterar recomendaciones generadas por el sistema IA fuera de los parámetros permitidos.

## 2. Rol: Turista organizado

Restricciones:
El usuario debe seleccionar paquetes previamente disponibles.
Algunas modificaciones del itinerario pueden estar limitadas.
El usuario no puede editar servicios incluidos por proveedores externos.
El usuario debe completar pagos para confirmar reservas.
El usuario no puede gestionar contenido de la plataforma.
El usuario depende de disponibilidad y horarios definidos por proveedores.
El usuario no puede acceder a estadísticas internas del sistema.
## 3. Rol: Viajero frecuente

Restricciones:
El usuario debe mantener una cuenta activa para conservar historial.
Algunas funciones rápidas requieren datos guardados previamente.
El usuario no puede modificar reservas ya confirmadas fuera de políticas permitidas.
El usuario no puede acceder a información de otros viajeros.
El sistema puede limitar reservas simultáneas excesivas.
El usuario no tiene permisos administrativos.
## 4. Rol: Turista digital

Restricciones:
El usuario requiere conexión a internet para funciones inteligentes.
Algunas funciones IA pueden depender de APIs externas.
El reconocimiento por voz puede variar según idioma o dispositivo.
El usuario no puede modificar configuraciones internas de IA.
Algunas traducciones automáticas pueden no ser totalmente precisas.
El usuario no tiene permisos para administrar contenido turístico.
## 5. Rol: Administrador de plataforma

Restricciones:
El administrador debe autenticarse con credenciales seguras.
Las acciones administrativas deben quedar registradas en auditorías.
El administrador no puede eliminar información crítica sin confirmación.
El administrador debe respetar políticas de privacidad y protección de datos.
El acceso administrativo puede estar limitado por niveles de permisos.
El administrador no puede modificar transacciones ya procesadas sin autorización especial.
El administrador depende del correcto funcionamiento del servidor y base de datos.
## 6. Rol: Proveedor de servicios turísticos

Restricciones:
El proveedor debe ser validado por la plataforma antes de publicar servicios.
El proveedor solo puede gestionar sus propios servicios.
El proveedor no puede modificar reservas ajenas.
El proveedor debe mantener información actualizada y verídica.
El proveedor depende de aprobación administrativa para ciertos contenidos.
El proveedor no puede acceder a estadísticas globales del sistema.
El proveedor debe respetar políticas de calidad y servicio de la plataforma.
## 7. Rol: Guía turístico

Restricciones:
El guía debe registrarse y validar su perfil profesional.
El guía solo puede gestionar sus propios tours y actividades.
El guía no puede acceder a configuraciones administrativas.
El guía depende de disponibilidad y reservas realizadas por usuarios.
El guía debe respetar horarios y condiciones establecidas.
El guía no puede modificar evaluaciones realizadas por viajeros.
El guía debe cumplir políticas de comportamiento y seguridad definidas por la plataforma.
